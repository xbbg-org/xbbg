## Requirements

- Python 3.10+
- A Bloomberg Terminal, B-PIPE, or ZFP environment that you are authorized to use
- Bloomberg C++ SDK/runtime components obtained from Bloomberg or another source you are authorized to use


For ZFP over leased lines, you also need Bloomberg-provided TLS client credentials and trust material. See [Configuration: ZFP over Leased Lines](/xbbg/python/reference/configuration/#zfp-over-leased-lines) for the exact `configure()` call and mutually exclusive connection options.

<Aside type="caution">
  xbbg does not provide Bloomberg subscriptions, credentials, entitlements, SDK licenses, or data rights. Use xbbg only with Bloomberg software, services, and data that your organization is authorized to use under its Bloomberg agreements and policies.
</Aside>

## Install xbbg

```bash
pip install xbbg
```

## Bloomberg SDK Setup

xbbg requires a Bloomberg C++ SDK shared library/runtime supplied by an authorized Bloomberg installation or package. You have several options:

<Tabs>
  <TabItem label="blpapi (Recommended)">
    Install the official Bloomberg Python bindings:
    ```bash
    pip install blpapi --index-url https://blpapi.bloomberg.com/repository/releases/python/simple/
    ```
    xbbg will automatically detect and use the SDK/runtime supplied by that `blpapi` installation.
  </TabItem>
  <TabItem label="Bloomberg Terminal">
    If you have Bloomberg Terminal installed, xbbg will auto-detect the DAPI SDK:
    - Windows: `C:\blp\DAPI`
    - Linux: `~/blp/DAPI` or `/opt/bloomberg/DAPI`
  </TabItem>
  <TabItem label="Manual SDK">
    For local development from this repo, use the repository helper to stage an authorized SDK locally:
    ```bash
    bash ./scripts/sdktool.sh
    ```

    Or set the `BLPAPI_ROOT` environment variable manually:
    ```bash
    export BLPAPI_ROOT=/path/to/blpapi_cpp_3.x.x.x
    ```
    Or configure at runtime:
    ```python
    import xbbg
    xbbg.set_sdk_path('/path/to/blpapi_cpp_3.x.x.x')
    ```
  </TabItem>
</Tabs>

## Verify Installation

```python
import xbbg

# Check SDK detection
print(xbbg.get_sdk_info())
# {'sources': [{'name': 'blpapi_python', 'version': '3.25.x.x', ...}], 'active': 'blpapi_python'}

# Check version
print(xbbg.__version__)
```

<Aside type="tip">
  Run `xbbg.get_sdk_info()` to see all detected SDK sources and which one is active.
</Aside>

## Install xbbg-mcp for coding agents

If you want Claude Code or OpenCode to send requests through your own locally configured Bloomberg environment, install the local MCP launcher and binary from GitHub Releases:

```bash
curl -fsSL https://raw.githubusercontent.com/alpha-xone/xbbg/main/scripts/install-xbbg-mcp.sh | sh
```

This installer currently targets **macOS arm64** and **Linux amd64**. Windows release `.zip` assets are attached to GitHub Releases for manual installation.

GitHub release assets include only the xbbg wrapper/binary pair. They do **not** bundle Bloomberg SDK files or the Bloomberg runtime. You must provide those from your own installed or otherwise authorized source, such as `blpapi` or `BLPAPI_ROOT`, subject to your Bloomberg agreements and entitlements.

The launcher uses the same Bloomberg runtime detection order as the Python package family:

1. `XBBG_MCP_LIB_DIR`
2. `BLPAPI_LIB_DIR`
3. `BLPAPI_ROOT`
4. locally staged authorized SDK under `vendor/blpapi-sdk/`
5. the official Python `blpapi` package

### Claude Code

```bash
claude mcp add --transport stdio xbbg -- ~/.local/bin/xbbg-mcp
```

### OpenCode

```json
{
  "$schema": "https://opencode.ai/config.json",
  "mcp": {
    "xbbg": {
      "type": "local",
      "command": ["/Users/you/.local/bin/xbbg-mcp"],
      "enabled": true
    }
  }
}
```

For the full env surface and local smoke test instructions, see [`apps/xbbg-mcp/README.md`](https://github.com/alpha-xone/xbbg/blob/main/apps/xbbg-mcp/README.md).
