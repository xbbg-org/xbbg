## Choose a package

| Package | Ecosystem | Install | Status | Use when |
|---------|-----------|---------|--------|----------|
| `xbbg` | Python / PyPI | `pip install xbbg` | Primary public package | You want the main supported API surface, DataFrame backends, notebooks, or Python services |
| `@xbbg/core` | Node.js / npm | `npm install @xbbg/core` | Experimental alpha | You need direct Node.js access to the shared Rust engine and Bloomberg request APIs |

<Aside type="note">
  Choose `xbbg` unless you specifically need Node.js. It is the main public package and the center of the documentation set.
</Aside>

## Packaging details that matter publicly

- `@xbbg/core` automatically resolves a supported platform package at install time for its native addon.
- Those platform-specific npm packages are implementation details of `@xbbg/core`, so they are not documented as separate products.
- Internal Rust crates, bindings, and private packages in the monorepo are intentionally omitted from this public docs tree.
