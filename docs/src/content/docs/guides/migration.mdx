# Migration from Legacy xbbg

xbbg 1.0 replaces the pure-Python 0.x implementation with a Rust-powered core. The public function
names (`bdp`, `bdh`, `bdib`, etc.) are unchanged, but output shapes, dependencies, and several
utility functions are not backward-compatible.

This guide documents every breaking change and provides concrete before/after examples.

<Aside type="caution">
  The `main` branch is the 1.0 release. If you need the exact 0.x behavior, pin to
  `release/0.x`. There is no shim layer that restores the old defaults — migration is required.
</Aside>

## Breaking Changes Summary

| Area | xbbg 0.x | xbbg 1.0 |
|------|----------|----------|
| Default output format | Wide (ticker as index / MultiIndex columns) | Long (ticker, field, value rows) |
| pandas | Required hard dependency | Optional — install separately if needed |
| Core dependencies | pandas | narwhals >= 1.30, pyarrow >= 22.0.0 |
| MultiIndex columns | `bdh` produced MultiIndex column tuples | Removed — use `pivot()` instead |
| Connection setup | `connect()` / `disconnect()` | `configure()` — engine auto-starts on first use |
| SDK version query | `getBlpapiVersion()` | `get_sdk_info()` — returns a dict |
| Package source path | `xbbg/` (in-tree) | `py-xbbg/src/xbbg/` |
| Python minimum | 3.8+ | 3.10+ |

## Output Format Migration

The largest behavioral change is the default output shape.

xbbg 0.x defaulted to **wide format**: `bdp` returned a DataFrame with tickers as the index and
field names as columns; `bdh` with multiple tickers returned a DataFrame with a two-level
MultiIndex on the columns axis `(ticker, field)`.

xbbg 1.0 defaults to **long format**: every function returns one row per (ticker, field)
observation with a `value` column. Long format is tidy data — it composes directly with
`groupby`, `merge`, and filter operations without unpacking MultiIndex levels.

### BDP Before and After

<Tabs>
  <TabItem label="0.x — wide default">
```python
from xbbg import blp

df = blp.bdp(
    ['AAPL US Equity', 'MSFT US Equity'],
    ['PX_LAST', 'VOLUME'],
)
# pandas DataFrame — ticker as index, fields as columns
#                  px_last    volume
# AAPL US Equity   185.50  45234521
# MSFT US Equity   378.91  78901234

price = df.loc['AAPL US Equity', 'px_last']
```
  </TabItem>
  <TabItem label="1.0 — long default">
```python
from xbbg import blp

df = blp.bdp(

)
# long format — ticker, field, value columns
#            ticker    field      value
# AAPL US Equity  PX_LAST     185.50
# AAPL US Equity   VOLUME   45234521
# MSFT US Equity  PX_LAST     378.91
```

```python
bars = blp.bdib('AAPL US Equity', dt='2024-01-15', interval=5)
```

```python
bars = blp.bdib('AAPL US Equity', dt='2024-01-15', interval=5)

bars = blp.bdib('AAPL US Equity', dt='2024-01-15', interval=5,
                format='semi_long')
# ticker          time                    open    high     low   close   volume
# AAPL US Equity  2024-01-15 09:30:00  185.00  185.80  184.90  185.60    12345
```

### Restoring Wide Output with pivot()

When downstream code requires wide layout, apply `.pivot()` after the request. The exact
parameter names differ between backends:

```python
from xbbg import blp

# --- narwhals / Polars backend (default) ---
df = blp.bdp(['AAPL US Equity', 'MSFT US Equity'], ['PX_LAST', 'VOLUME'])
wide = df.pivot(on='field', index='ticker', values='value')

# BDH with narwhals / Polars
hist = blp.bdh('AAPL US Equity', ['open', 'high', 'low', 'close'],
               '2024-01-01', '2024-01-31')
hist_wide = hist.pivot(on='field', index=['ticker', 'date'], values='value')

# --- pandas backend ---
df_pd = blp.bdp(['AAPL US Equity', 'MSFT US Equity'], ['PX_LAST', 'VOLUME'],
                backend='pandas')
wide_pd = df_pd.pivot(columns='field', index='ticker', values='value')
```

### SEMI_LONG: Closest to Old Wide

`format='semi_long'` returns one row per ticker with each field as a column. For `bdp`, this is
the shape closest to the 0.x default:

```python
from xbbg import blp

df = blp.bdp(
    ['AAPL US Equity', 'MSFT US Equity'],
    ['PX_LAST', 'VOLUME'],
    format='semi_long',
)
#            ticker  px_last     volume
# AAPL US Equity   185.50   45234521
# MSFT US Equity   378.91   78901234
```

### Format Reference

| Format | Shape | Supports lazy backends |
|--------|-------|------------------------|
| `long` (default) | ticker, [date,] field, value | Yes |
| `long_typed` | ticker, [date,] field, value_f64, value_i64, … | Yes |
| `long_with_metadata` | ticker, [date,] field, value, dtype | Yes |
| `semi_long` | ticker, [date,] then one column per field | Yes |

## Backend Migration

### Before: pandas Only

xbbg 0.x depended on pandas. Every function returned `pd.DataFrame`. No configuration was needed.

### After: narwhals Default, pandas Optional

xbbg 1.0 uses PyArrow for data transport and narwhals for DataFrame construction. pandas is not
installed automatically.

**Core dependencies** (installed with xbbg):
- `narwhals >= 1.30`
- `pyarrow >= 22.0.0`

**Optional backends** (install separately):
```bash
pip install pandas    # pandas DataFrames
pip install polars    # Polars DataFrames
pip install duckdb    # DuckDB relations
```

### Quick Compat: backend='pandas'

Pass `backend='pandas'` to restore `pd.DataFrame` output. Combine with `format='semi_long'`
to also restore the one-row-per-ticker shape:

```python
from xbbg import blp

# pd.DataFrame, long format (ticker, field, value columns)
df = blp.bdp('AAPL US Equity', ['PX_LAST', 'VOLUME'], backend='pandas')

# pd.DataFrame, semi_long (one row per ticker, fields as columns)
df = blp.bdp('AAPL US Equity', ['PX_LAST', 'VOLUME'],
             backend='pandas', format='semi_long')
```

To apply `backend='pandas'` globally for a session:

```python
from xbbg import set_backend

set_backend('pandas')
# All subsequent calls return pd.DataFrame
```

### Checking What Is Available

```python
from xbbg import get_available_backends, print_backend_status

print(get_available_backends())   # e.g. ['narwhals', 'pyarrow', 'pandas']
print_backend_status()            # installed/missing with version info
```

## API Changes

### Renamed and Replaced Functions

| xbbg 0.x | xbbg 1.0 | Notes |
|----------|----------|-------|
| `connect(host, port)` | `configure(server_host=…, server_port=…)` | Engine auto-starts — `configure()` only needed for non-default settings |
| `disconnect()` | `shutdown()` / `reset()` | Rarely needed; engine lifecycle is automatic |
| `getBlpapiVersion()` | `get_sdk_info()` | Returns `dict` with SDK paths and `runtime_version` |
| `fieldInfo(fields)` | `bflds(fields=fields)` | `fieldInfo` remains as an alias |
| `fieldSearch(keyword)` | `bflds(search_spec=keyword)` | `fieldSearch` remains as an alias |
| `lookupSecurity(name)` | `blkp(name)` | Renamed for API consistency |
| `asset_config(asset)` | `market_info(ticker)` | **Removed in 1.0** — use `market_info(ticker)` |

### Connection Setup

`connect()` and `disconnect()` no longer exist. The engine starts automatically on first use.
Call `configure()` once before your first request only if you need a non-default host, port,
or authentication method:

<Tabs>
  <TabItem label="0.x">
```python
from xbbg import blp

blp.connect(host='192.168.1.100', port=18194)

df = blp.bdp('AAPL US Equity', 'PX_LAST')

blp.disconnect()
```
  </TabItem>
  <TabItem label="1.0">
```python
from xbbg import blp

# configure() replaces connect() — call once before first request
blp.configure(server_host='192.168.1.100', server_port=18194)

df = blp.bdp('AAPL US Equity', 'PX_LAST')

# No disconnect needed — engine lifecycle is managed automatically
```
  </TabItem>
</Tabs>

Connecting to `localhost:8194` (the default Bloomberg Terminal port) requires no
`configure()` call at all.

### SDK Version

```python
# 0.x
version_str = blp.getBlpapiVersion()   # str, e.g. "3.22.1.1"

# 1.0
info = blp.get_sdk_info()              # dict
print(info['active'])                  # active SDK path
print(info['runtime_version'])         # linked C SDK version, e.g. "3.26.2.1"
```

### Field Metadata

`fieldInfo()` and `fieldSearch()` are now aliases for the unified `bflds()` function.
Both aliases continue to work, but new code should use `bflds()` directly:

```python
from xbbg import blp

# Aliases — still work in 1.0
blp.fieldInfo(['PX_LAST', 'VOLUME'])
blp.fieldSearch('vwap')

# Canonical 1.0 form
blp.bflds(fields=['PX_LAST', 'VOLUME'])   # metadata for specific fields
blp.bflds(search_spec='vwap')             # keyword search
```

## Removals

### Format.WIDE

`Format.WIDE` was **removed** in 1.0. Passing `format='wide'` or
`format=Format.WIDE` raises a `ValueError`. Use the default long format
and pivot afterwards:

```python
df = blp.bdp('AAPL US Equity', 'PX_LAST')
df_wide = df.pivot(on='field', index='ticker', values='value')
```

Alternatively, use `format='semi_long'` which produces one column per field
while keeping tickers in a column.

### asset_config()

`asset_config()` was removed in 1.0. Use `market_info(ticker)` to retrieve
market and exchange metadata from Bloomberg directly:

```python
from xbbg.markets import market_info
info = market_info('ES1 Index')
# Returns dict with 'exch', 'tz', session times, and related fields
```

## Step-by-Step Upgrade

<Steps>

1. **Upgrade the package**

   ```bash
   pip install --upgrade xbbg
   ```

2. **Verify the Rust backend loads**

   ```python
   import xbbg
   print(xbbg.get_sdk_info())
   ```

   If this raises an `ImportError`, your Bloomberg C++ SDK may not be discoverable.
   Set `BLPAPI_ROOT` to the extracted SDK directory and retry.

3. **Install optional dependencies**

   ```bash
   pip install pandas   # if your code needs pd.DataFrame output
   pip install polars   # if you want Polars output
   ```

4. **Set a session-level backend default (optional)**

   Add one line at the top of your script to restore pandas output everywhere:

   ```python
   from xbbg import set_backend
   set_backend('pandas')
   ```

5. **Remove connect() / disconnect() calls**

   Delete all `blp.connect(...)` and `blp.disconnect()` calls. If you were passing a
   non-default host or port to `connect()`, replace those calls with a single
   `blp.configure(server_host=..., server_port=...)` call before your first request.

6. **Handle output format changes**

   The safest incremental path is to add `format='semi_long'` to each `bdp` call and
   `format='semi_long'` (or explicit `pivot()`) to each `bdh` call, then migrate
   downstream code to long format call-by-call.

7. **Replace deprecated function names**

   Use the [API changes table](#renamed-and-replaced-functions) above. `connect()`,
   `getBlpapiVersion()`, and the MultiIndex output shape have no silent aliases — code
   that calls them will raise `AttributeError`.

</Steps>
